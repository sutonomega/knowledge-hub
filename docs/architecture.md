# Architecture

## システム構成

```text
┌──────────────────────────┐
│         Browser          │
│  (PC / Smartphone)       │
└────────────┬─────────────┘
             │ HTTP / HTTPS
┌────────────▼─────────────┐
│      Frontend (Next.js)  │
└────────────┬─────────────┘
             │ API
┌────────────▼─────────────┐
│      Backend (Next.js)   │
├──────────────────────────┤
│ Markdown Parser          │
│ Search Engine            │
│ Tag Manager              │
│ Link Manager             │
│ Media Manager            │
└────────────┬─────────────┘
             │
     ┌───────┴────────┐
     │                │
┌────▼────┐     ┌─────▼─────┐
│Database │     │ File Store │
│SQLite   │     │Media Files │
└─────────┘     └────────────┘
```

---

# 構成

## Frontend

ユーザーインターフェースを担当する。

### 主な役割

- ノート一覧
- Markdownエディタ
- 閲覧モード
- タグ表示
- 検索画面
- メディア表示
- AI画面（将来）

---

## Backend

データ管理と各種処理を担当する。

### 主な役割

- ノート管理
- タグ管理
- リンク解析
- バックリンク生成
- 全文検索
- メディア管理
- AI連携（将来）

---

## Database

メタデータのみ保存する。

例

- ノート情報
- タグ
- リンク情報
- 更新日時
- 添付ファイル情報

---

## File Store

実際のデータを保存する。

保存対象

- Markdown
- 画像
- 動画
- 音声
- PDF
- その他添付ファイル

---

# データの考え方

データは可能な限り標準形式で保持する。

| 種類   | 形式             |
| ------ | ---------------- |
| ノート | Markdown (.md)   |
| 画像   | png / jpg / webp |
| 動画   | mp4 / webm       |
| 音声   | mp3 / wav        |
| 文書   | pdf              |
| その他 | 任意ファイル     |

---

# 基本設計方針

- データをユーザーが管理できる構造にする
- フロントエンドとバックエンドを分離し、拡張しやすくする
- APIを中心とした設計にする
- 将来的なAI連携を考慮した構成にする
- 特定サービスに依存しない設計を目指す
